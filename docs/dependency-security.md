# Development dependency security

- Status: Current maintenance policy
- Last reviewed: 2026-07-18
- Scope: Composer and Node.js development dependencies; the WordPress plugin
  release package contains neither dependency tree

## Runtime boundary

`package.json` contains development dependencies only. The release gate excludes
`node_modules`, Composer `vendor`, source maps, tests, and development metadata
from the plugin ZIP. `pnpm audit --prod` and `composer audit --locked` therefore
remain separate evidence from the full development-tooling audit.

## Policy

- High and critical advisories in the resolved development tree must be zero.
- Exact pnpm overrides may patch a transitive dependency when the change is
  compatible with the current WordPress package snapshot and the full release
  gate passes.
- Do not apply the open-ended ranges generated by `pnpm audit --fix`. They may
  cross major versions; the 2026-07-18 trial selected Babel 8 and prevented all
  JavaScript tests from starting.
- Moderate and low advisories are reviewed but are not a hard gate when the only
  fix requires an unvalidated major upgrade or no upstream patch exists.
- Update the direct WordPress packages as one tested snapshot. Do not move an
  individual `@wordpress/*` package to `latest` merely to silence an unrelated
  transitive advisory.

The scheduled Dependency Audit workflow and dependency-related pull requests run
`pnpm audit --audit-level=high` and `composer audit --locked`. A newly reported
high or critical issue fails that workflow until it is fixed or the project
records a narrower, evidence-based policy change.

## Verification required for override changes

1. `pnpm install --frozen-lockfile` succeeds with the pinned pnpm version.
2. `pnpm check` passes all TypeScript, JavaScript, ESLint, and Stylelint checks.
3. The production build leaves the tracked `build/` output unchanged unless the
   pull request intentionally changes shipped code.
4. Run `bash scripts/release-gate.sh` in an isolated branch or worktree.
5. Record the remaining audit levels and modules in the pull request rather than
   hiding warnings.

An isolated 2026-07-18 verification reduced the development audit from 3
critical, 44 high, 40 moderate, and 4 low dependency paths to 0 critical, 0
high, 11 moderate, and 1 low. The remaining findings are transitive through the
official WordPress development snapshot and are not included in the plugin ZIP.
