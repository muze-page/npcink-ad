# Magick AD Compatibility Guide

目标：让插件在不同主题、缓存、设备与同意插件环境下稳定运行。

## 1. 推荐基线配置

- 展示级别：`简洁`（生产环境）
- 创意优先：`代码/HTML`、`图片`、`视频`（可视化模板仅在高级场景）
- 统计写入：`async`
- 缓存友好 Slot 轮播：`开启`
- Full HTML 沙箱：`开启`
- 去重策略：`session`（默认）

## 2. 缓存与随机展示

全页缓存场景下，服务端随机会被缓存放大。建议：

1. 开启 `缓存友好 Slot 轮播（客户端选择）`。
2. 去重策略使用 `session` 或 `cookie`（已同意时）。
3. 观察系统设置中的队列积压与最久等待时间。

## 3. 同意（CMP）适配

### 3.1 内置自动识别

插件会识别常见信号（如 Cookiebot、OneTrust、Complianz、CookieYes、Borlabs、Moove 等）并通过统一入口判断同意状态。

### 3.2 自定义接入（优先级最高）

```php
add_filter('magick_ad_has_consent', function ($has_consent) {
    // 你的同意系统判断逻辑
    return $has_consent;
});
```

### 3.3 关键原则

- 启用“需要同意后统计”时，未同意将不写统计。
- 同意恢复后，追踪自动恢复。

## 4. 统计可靠性与降级

当数据库抖动或高峰写入失败时，插件会：

1. 优先写入异步队列；
2. 队列不可用时尝试回退队列；
3. Cron 任务自动回收回退队列。

建议持续关注：
- 兼容报告中的风险项
- Site Health 中 `统计队列积压` 与 `Cron 任务健康度`

## 5. 模板兼容策略

- 系统模板与用户模板均带 `templateVersion`。
- 老模板会在迁移阶段自动补齐版本和字段映射。
- 模板导入导出优先使用最新 schema。

## 6. 兼容报告使用方式

1. 打开 `wp-admin/admin.php?page=magick-ad-compat`
2. 点击 `刷新报告`
3. 导出 `JSON` 或 `Markdown` 给运维/测试/客户

报告内容包括：
- 环境信息（WP/PHP/MySQL/缓存）
- 运行设置（追踪、签名、同意、写入模式）
- 体检检查项（节点/Cron/队列/同意）
- 风险与建议动作
