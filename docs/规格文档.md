# WordPress 广告插件规格文档 v0.1

- 文档版本：v0.1（Draft）
- 日期：2026-01-28
- 目标平台：WordPress（Block Editor/Gutenberg）
- 本文重点：**弹窗广告（Popup）**与**横栏广告（Bar）**的 Token 体系、UI 分层、模板库（预设 + 自定义模板）与可扩展架构

---

## 1. 背景与目标

你计划提供 5 类广告：**HTML、图片、视频、弹窗、横栏**，并持续新增更多类型。每种类型都希望提供大量个性化参数（颜色/位置/大小/动画等），同时做到：

- 对新手：**简单好用**（少量关键设置即可发布）
- 对专业用户：**深度可控**（Token、模板、条件、行为、样式扩展）
- 对未来：**可扩展**（新增广告类型不推翻体系）

为达成以上目标，本插件采用：

- **Block Editor 作为内容编辑器**（广告内容像做页面一样做）
- **模板库 = Block Patterns + Variations + Block Styles**
- **用户自定义模板 = Synced Patterns**
- **个性化 = 设计令牌（Design Tokens）+ 渐进式披露（Quick/Design/Expert）**

相关 WordPress 能力依据：Patterns、Variations、Block Styles、Synced Patterns、InnerBlocks、Block Supports、theme.json。([WordPress Developer Resources][1])

---

## 2. 范围定义

### 2.1 本期（v1）必须交付（MVP+）

**核心**

1. 广告管理（创建/编辑/启用/停用/复制/归档）
2. 两种广告类型优先落地：**弹窗**、**横栏**（最能体现“模板 + 行为 + Token”）
3. 模板系统：预设模板 + 用户自定义模板（Synced Pattern）
4. 行为系统：触发 + 频控 + 关闭规则
5. 投放系统：基础定向（页面/文章类型/分类/设备/登录状态）
6. 调试：为什么不展示（命中/未命中原因）

**合规/生态护栏（必须）**

- 禁止未经同意追踪/回传数据；如有外部通信必须显式 opt-in。([WordPress Developer Resources][2])
- WordPress.com 生态：避免在 WP-Admin/商店界面投放插件广告（商业版也建议遵守）。([developer.wordpress.com][3])

### 2.2 后续版本（v1.1+）

- HTML 广告：高级净化与权限模型
- 图片/视频广告：更多媒介能力、懒加载、替换素材
- A/B 测试、排期、多尺寸适配、导入导出、多站点模板分发等

### 2.3 明确不做（v1 不做）

- 自研 CMP（同意弹窗）——只做 **“同意信号接入 + 门控”**
- 广告竞价/瀑布流/复杂广告联盟对接（可留扩展点）

## 3. 关键概念与术语

> 关键是把“个性化参数爆炸”拆层，避免每种广告类型都变成一个巨大的表单。

- **Creative（内容）**：广告文案、图片、按钮、视频等，由 Block Editor 编辑
- **Container（容器/类型）**：弹窗/横栏等展示容器，负责 DOM 结构与布局约束
- **Behavior（行为）**：触发方式、频控、关闭方式、动画时机
- **Placement（投放/插入）**：放在哪、对谁展示、什么时候展示
- **Template（模板）**：预设布局（Pattern）、预设配置入口（Variation）、视觉皮肤（Block Style）
- **Token（设计令牌）**：把 UI 控件收敛成稳定的 CSS 自定义属性（CSS Variables），跨类型复用

## 4. 总体架构

### 4.1 管理端（WP-Admin）

- 广告列表页（CPT）：创建、复制、启用/停用、快速预览、状态（草稿/发布/归档）
- 广告编辑页：
  - Block Editor 编辑 Creative
  - 右侧 Inspector：Container / Behavior / Placement / Tokens（Quick/Design/Expert 分层）
  - 模板库（内置 Patterns/Variations/Styles）
  - 用户模板：Synced Patterns 管理与复用（“编辑一次，全站同步”）([WordPress.org][4])

### 4.2 前端渲染

- 统一渲染器（Renderer）
  - 读取广告实体（CPT + meta）
  - 计算 Placement/Targeting 命中
  - 输出占位（防 CLS）、容器结构、Creative 内容
  - 注入 token（CSS variables）与必要 JS（行为、触发、频控）

- 弹窗：前端 JS 控制显示/隐藏、焦点管理（A11y）
- 横栏：可选 sticky 或 push-down（推开内容）

## 5. 数据模型与存储

### 5.1 存储形式

- 使用 **自定义文章类型（CPT）**：`mad_ad`（示例）
- 主要字段：
  - `post_title`：广告名称
  - `post_content`：Block 内容（Creative）
  - `post_status`：draft/publish

- Meta（JSON）：
  - `mad_type`：`popup | bar | html | image | video`
  - `mad_tokens`：设计令牌集合（见第 7 节）
  - `mad_behavior`：触发/频控/关闭规则
  - `mad_placement`：投放规则（页面/设备/角色/排除等）
  - `mad_version`：配置 schema 版本号（用于迁移）

> 说明：模板（pattern）尽量不存 “模板 HTML”，而存 “Block 结构 + tokens/behavior 默认值”。Patterns/Variations/Styles 用于创建时初始化；之后用户改动落在 post_content + meta。

### 5.2 配置 schema 示例（简化）

```json
{
  "type": "popup",
  "schemaVersion": 1,
  "tokens": {
    "color.bg": "#ffffff",
    "color.primary": "#2563eb",
    "radius": "12px",
    "shadow": "md",
    "width.max": "520px"
  },
  "behavior": {
    "trigger": { "kind": "delay", "ms": 1500 },
    "frequency": { "scope": "session", "max": 1 },
    "close": { "showX": true, "esc": true, "overlayClick": true }
  },
  "placement": {
    "pages": { "include": ["all"], "exclude": [] },
    "device": ["desktop", "mobile"]
  }
}
```

## 6. 编辑器与模板系统规格

### 6.1 采用 WordPress 原生模板能力（必须）

1. **Block Patterns**：作为“模板库（结构预设）”，插入后可继续编辑。([WordPress Developer Resources][1])
2. **Block Variations**：作为“同一个容器块的预配置入口”（例如 Popup 的居中/右下角滑入）。([WordPress Developer Resources][5])
3. **Block Styles**：作为“外观皮肤”（圆角/阴影/边框风格）。([WordPress Developer Resources][6])
4. **Synced Patterns**：作为“用户自定义模板”，编辑一次，全站引用同步更新。([WordPress.org][4])
5. **InnerBlocks**：容器块内部允许嵌套内容，提供内容骨架模板并可限制 allowed blocks。([WordPress Developer Resources][7])

### 6.2 Block Supports + theme.json 对齐（降低你自建控件量）

- 容器块应尽量启用 block supports（颜色、间距、排版等），让用户用 WordPress 原生 UI 调整，减少“你自己造的颜色选择器/间距控件”。([WordPress Developer Resources][8])
- 同时尊重主题的全局风格系统 `theme.json`（调色板、字号、间距预设等），实现“广告风格融入主题”。([WordPress Developer Resources][9])

## 7. 设计令牌（Tokens）体系

### 7.1 Token 设计原则

- **稳定**：新增能力只新增 token，不改旧 token 语义
- **跨类型复用**：颜色/圆角/阴影/间距等通用 token 在 popup/bar 通用
- **映射到 CSS Variables**：最终输出为 `--mad-*`，同时保留抽象 token key（便于后续迁移）
- **渐进披露**：Quick 模式只暴露 6–10 个 token；Design/Expert 才展开细项

### 7.2 通用 Token 清单（v1）

> token key → CSS var → 说明（默认值）

**颜色**

- `color.bg` → `--mad-bg` → 背景色（默认：theme 背景/白）
- `color.text` → `--mad-text` → 文本色
- `color.primary` → `--mad-primary` → 主色（按钮/强调）
- `color.border` → `--mad-border` → 边框色
- `overlay.color` → `--mad-overlay` → 遮罩色（popup 用）

**布局与尺寸**

- `radius` → `--mad-radius` → 圆角（8px/12px）
- `shadow.level` → `--mad-shadow` → 阴影等级（none/sm/md/lg）
- `spacing.padding` → `--mad-pad` → 内边距
- `zIndex` → `--mad-z` → 层级（默认高于 header）

**动画**

- `anim.enter` → `--mad-anim-enter` → 进入动画（fade/slide/zoom）
- `anim.duration` → `--mad-anim-ms` → 动画时长（200–300ms）
- `motion.reduce`（布尔）→ 自动遵守 prefers-reduced-motion（实现层）

## 8. 广告类型规格（v1 重点：Popup & Bar）

> 其他类型（HTML/图片/视频）在 v1.1+ 完善；本节给出框架与最低要求，避免未来推翻。

### 8.1 弹窗广告（Popup）规格

#### 8.1.1 容器块

- Block：`mad/popup`
- 内部内容：`InnerBlocks`（建议允许：paragraph/heading/image/buttons/list，禁止复杂嵌套布局以免破坏弹窗体验）([WordPress Developer Resources][7])
- Variation（预配置入口，v1 至少 6 个）([WordPress Developer Resources][5])
  1. `popup-center`：居中模态
  2. `popup-slide-rb`：右下角滑入
  3. `popup-slide-lb`：左下角滑入
  4. `popup-top`：顶部通知式弹出（更轻量）
  5. `popup-fullscreen`：全屏引导
  6. `popup-minimal`：极简无图版本

#### 8.1.2 Popup 专属 Tokens

- `popup.width.max` → `--mad-popup-maxw`（520px）
- `popup.width.min` → `--mad-popup-minw`（280px）
- `popup.position` → `--mad-popup-pos`（center/rb/lb/top）
- `popup.offset.x/y` → `--mad-popup-offx/offy`（角落滑入时开放）
- `overlay.opacity` → `--mad-overlay-opacity`（0.4）
- `close.size` → `--mad-close-size`（32px）
- `close.position` → `--mad-close-pos`（top-right 默认）

#### 8.1.3 行为（Behavior）

**触发（Trigger）**（v1 支持 3 种）

- delay：延迟 ms
- scroll：滚动到百分比
- click：点击某个触发按钮（由短代码/块生成触发器）

**频控（Frequency）**

- scope：`session | day | week`
- max：次数（默认 session=1）
- 存储：cookie/localStorage（实现时确保可被站点隐私策略控制；未同意时禁用追踪型频控，或仅使用必要存储）

**关闭规则（Close）**

- showX：默认 true
- ESC：默认 true（键盘可用性）
- overlayClick：默认 true（可在 Design 模式关闭）

#### 8.1.4 SEO/体验护栏（必须）

- 提供“温和模式”预设：移动端不首屏立即弹、避免遮挡主要内容（作为模板推荐语）
- 在文档/设置中提示：侵入式插屏可能影响用户体验与搜索表现（不做恐吓，但要提示）。([Google for Developers][10])

#### 8.1.5 无障碍（A11y）强制要求

弹窗属于**模态对话框**：

- 打开后：焦点必须进入弹窗内部
- Tab/Shift+Tab：焦点不能跳出弹窗（focus trap）
- 关闭后：焦点返回触发点
- 键盘：ESC 关闭（若启用）
  这些属于 WAI-ARIA 模态对话框模式的核心要求。([W3C][11])

---

### 8.2 横栏广告（Bar）规格

#### 8.2.1 容器块

- Block：`mad/bar`
- 内部内容：InnerBlocks（允许更轻量：paragraph/buttons/image）
- Variation（v1 至少 6 个）([WordPress Developer Resources][5])
  1. `bar-top-sticky`：顶部吸顶
  2. `bar-bottom-sticky`：底部吸顶
  3. `bar-top-pushdown`：顶部推开内容（占位）
  4. `bar-bottom-pushup`：底部推开内容
  5. `bar-floating`：浮层不占位（谨慎用，默认不推荐）
  6. `bar-minimal`：极简一行 + 按钮

#### 8.2.2 Bar 专属 Tokens

- `bar.height` → `--mad-bar-h`（56px）
- `bar.position` → `--mad-bar-pos`（top/bottom）
- `bar.behavior` → `--mad-bar-mode`（pushdown/sticky/floating）
- `bar.padding` → `--mad-bar-pad`（12–16px）
- `bar.close.show`（布尔，默认 true）

#### 8.2.3 性能护栏（CLS 必须）

- 对 pushdown 模式：必须预留空间/稳定高度，避免加载后把页面内容顶来顶去
- 对 sticky/floating：也要避免动态改变高度造成抖动
  “预留足够空间是避免布局偏移的最可靠方式”，尤其是广告位。([Google for Developers][12])

## 9. UI/UX 分层（关键：既个性化又简单）

### 9.1 编辑页右侧面板结构

统一分 4 组（对所有广告类型一致）：

1. **类型与模板**（Type & Template）
2. **内容**（Content：Block Editor 区域）
3. **行为**（Behavior）
4. **投放**（Placement/Targeting）
5. **样式**（Style/Tokens）

### 9.2 三档模式（渐进式披露）

> 默认只显示 Quick；其余折叠，避免把用户吓退。

- **Quick（默认）**：
  - 模板选择（variation/pattern）
  - 主色（primary）、背景（bg）
  - 位置（popup：center/rb/lb；bar：top/bottom）
  - 触发（delay/scroll/click）
  - 频控（session 1 次）
  - 关闭（X/ESC）

- **Design（进阶）**：
  - 尺寸（max width/height）
  - 圆角/阴影/遮罩透明度
  - 动画类型/时长
  - 移动端单独样式（可选）

- **Expert（专家）**：
  - 自定义 CSS（仅 token 范围或 scoped CSS，避免全站污染）
  - 更复杂触发（后续版本）
  - 开发者 Hook（见第 12 节）

## 10. 内置模板库（交付清单）

> 模板=“场景驱动”，不要按“样式驱动”。用户更容易选对。

### 10.1 Popup 内置 Pattern（v1 建议 10 个）

每个模板应提供：缩略图、用途说明、推荐触发/频控默认值、移动端建议。

1. **限时折扣（带倒计时）**：标题 + 倒计时 + CTA 按钮
2. **优惠券领取**：券码 + 复制按钮 + CTA
3. **Newsletter 订阅**：输入框 + 订阅按钮（可选 Mailchimp 后续）
4. **下载资料（Lead Magnet）**：图标 + 2 行说明 + 下载按钮
5. **新功能公告**：轻量文本 + “了解更多”
6. **退出挽回（Exit intent 预留）**：小幅折扣 + 关闭弱化（后续行为支持）
7. **全屏引导**：大图 + 分步说明 + CTA
8. **视频介绍**：封面图 + 播放按钮（视频广告能力复用）
9. **社会证明（评分/评价）**：星级 + 引用短评 + CTA
10. **极简提示**：一行标题 + 一个按钮（尽量不侵入）

> 注：倒计时、表单等可先用简化块实现，复杂交互后续增强。

### 10.2 Bar 内置 Pattern（v1 建议 10 个）

1. **顶部促销条**：一句话 + CTA
2. **底部订阅条**：一句话 + 输入框（可选）
3. **免运费提醒**：进度提示（后续可接购物车）
4. **Cookie/隐私提示条**（轻提示，非 CMP）
5. **站点公告条**：维护/升级通知
6. **App 下载引导**：图标 + 按钮
7. **新用户欢迎条**：短提示 + CTA
8. **直播/活动开始提醒**：时间 + “立即观看”
9. **免责声明/披露条**：联盟链接披露/赞助披露（文案模板）
10. **极简纯文本条**：不带按钮

## 11. 性能与 Web Vitals 规范（必须项）

### 11.1 CLS（布局稳定性）

- 所有 in-flow 广告位（pushdown bar、正文插入型未来扩展）必须 **预留空间**
- 广告位高度/宽度尽量稳定；响应式用 `min-height` 或 `aspect-ratio` 策略
  Web.dev 与 Google Publisher Tag 都强调：**预留空间是避免布局偏移最可靠的方式**。([web.dev][13])

### 11.2 加载策略

- 首屏可见广告：优先加载，但要避免阻塞页面关键渲染路径
- 非首屏：懒加载（IntersectionObserver）
- 任何第三方脚本：可配置延迟加载并提供调试开关
  广告加载对页面速度的影响与 CLS 风险需要明确控制。([web.dev][14])

## 12. 安全、合规与生态约束（强制）

### 12.1 插件生态规则（强制）

- **不得未经用户同意追踪/联系外部服务器**；如需要外部通信必须显式 opt-in，并在说明中披露。([WordPress Developer Resources][2])
- WordPress.com 生态强调不应在 WP-Admin/商店出现插件广告。([developer.wordpress.com][3])

### 12.2 内容安全（HTML/自定义代码）

- HTML 广告（后续）：必须提供净化策略
- 参考：`wp_kses_post()` 可用于按允许标签集合净化内容（注意它面向“文章内容”，不等同于所有场景都安全，需按权限/场景加严）。([WordPress Developer Resources][15])
- 所有输入：先验证再净化；所有输出：按上下文 escape
  官方安全章节强调数据验证/净化的重要性。([WordPress Developer Resources][16])

### 12.3 接口与权限

- 若提供 REST API：必须设置 `permission_callback` 并进行鉴权（可配合 nonce）。([WordPress Developer Resources][17])

### 12.4 同意信号接入（建议做成可插拔）

如果用户使用 Google 广告：

- EEA/UK 自 2024-01-16 起、瑞士自 2024-07-31 起，投放个性化广告需要 Google 认证且集成 IAB TCF 的 CMP。([Google 帮助][18])
- TCF v2.2 对 UI 透明度与撤回同意易达性提出要求（例如首层披露 vendor 数量、易于重新打开 CMP 并撤回）。([IAB Europe][19])
- Google Consent Mode 用于传递广告/分析等 consent 信号（ad_storage、ad_user_data、ad_personalization 等）。([Google 帮助][20])

> v1 目标：提供“门控接口”，不做 CMP；能读取/接入外部 CMP/Consent Mode 信号，决定是否加载某些广告/脚本。

## 13. 扩展性设计（为更多广告类型预留）

### 13.1 类型插件化

- 每个广告类型实现一个 `AdTypeModule`：
  - `get_type_key()`
  - `get_default_tokens()`
  - `get_variations() / get_patterns() / get_styles()`
  - `render_frontend()`
  - `validate_config()`

- 新增类型不修改核心渲染器逻辑，只注册模块

### 13.2 Token 向后兼容与迁移

- `schemaVersion` 必须存在
- 升级时：
  - 新增 token：补默认值
  - 废弃 token：保留但忽略（或映射到新 token）
  - 提供迁移日志（调试用）

## 14. 调试与可观测性（建议 v1 就做）

### 14.1 “为什么没展示”调试面板

- 当前页面命中广告列表
- 对每条广告显示：
  - 命中/未命中原因（设备、页面规则、频控、未同意等）
  - 最终使用的 tokens 摘要
  - 渲染耗时（简要）

### 14.2 预览模式

- 管理端预览：模拟不同设备、不同触发、重置频控

## 15. 测试验收标准（v1）

### 15.1 功能验收

- Popup/Bar 6 个 variation 可用
- 每类至少 10 个 pattern 可插入并可编辑
- Synced pattern 可作为用户模板复用且修改能同步生效。([WordPress.org][4])

### 15.2 性能验收

- pushdown bar 不产生明显 CLS（预留空间）
- popup 不阻塞首屏渲染（默认延迟触发）
- 重要位置广告位加载策略符合“减少布局偏移”的最佳实践（预留空间）。([Google for Developers][12])

### 15.3 A11y 验收（Popup）

- 键盘可达：Tab 不逃逸、ESC 可关闭、关闭后焦点返回
  符合 WAI-ARIA 模态对话框模式关键点。([W3C][11])

### 15.4 安全验收

- 所有保存操作：capability check + nonce
- 所有输入：验证/净化；输出 escape（尤其是 HTML 广告后续）
- REST API（若有）：permission_callback 正确鉴权。([WordPress Developer Resources][17])
