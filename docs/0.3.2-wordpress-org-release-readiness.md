# Npcink Ad 0.3.2 WordPress.org 首发候选记录

- 状态：Release Candidate
- 日期：2026-07-18
- 发布渠道：GitHub Release 与 WordPress.org 首次提交

## 目标与非目标

0.3.2 的目标是把已经通过真实使用的单 Promotion 工作流整理成可审核、可翻译、
可从同一产物发布到 GitHub 和 WordPress.org 的首发候选版本。

本版本不增加广告类型、位置、统计、追踪、Popup、悬浮/吸顶、频控、地理定向、
第二套数据模型或新的一级管理页面，也不改变现有 Promotion 数据和投放判定。

## 变更边界

- 插件头、`NPCINK_AD_VERSION`、`package.json`、PHPStan bootstrap、根目录
  `readme.txt` Stable tag、Git 标签和 WordPress.org SVN 标签统一为 `0.3.2`。
- 根目录 `readme.txt` 收口到 WordPress.org 建议的简明目录说明；历史版本移入
  `changelog.txt`。
- 新增项目自有的英文/简体中文图标、横幅和三组真实界面截图，源 SVG 保留在
  `docs/wordpress-org-submission/source/`；目录 PNG 不进入插件 ZIP。
- 删除 WordPress.org 托管插件不需要的手动 `load_plugin_textdomain()` 调用。
  PHP 简体中文通过 `wp-content/languages/plugins/` 标准语言包路径加载，编辑器
  继续验证完整 JSON 目录；仓库继续维护完整 POT、PO、MO 和 JSON 源文件。
- 发布包新增根目录 `changelog.txt`，继续排除 `.github`、测试、文档、开发依赖、
  源映射和源码编辑器入口。

## 发布门禁

以下门禁全部通过后才能创建 `v0.3.2` 标签：

1. `composer check`：PHPUnit、PHPCS、PHPStan；
2. `pnpm check`：TypeScript、Jest、ESLint、Stylelint；
3. `bash scripts/release-gate.sh`：版本契约、生产构建、翻译完整性、体积预算、
   ZIP 内容、SHA-256 和官方 Plugin Check；
4. WordPress Playground：最低 WP 6.5/PHP 8.1 与当前 WordPress/PHP 组合；
5. 经典主题与区块主题矩阵；
6. 打包插件 Gutenberg E2E；
7. `git diff --check` 与最终 staged diff 审查。

Plugin Check 中 `load_plugin_textdomain` 警告必须消失。允许保留但必须说明的两条
warning 是受控 Promotion `meta_query`，以及授权真实页面预览使用的第三方缓存
约定常量 `DONOTCACHEPAGE`；任何 error 都阻断发布。

## 单一产物规则

`v0.3.2` 标签工作流生成的 `npcink-ad-0.3.2.zip` 与对应 `.sha256` 是唯一发布
产物。GitHub Release、WordPress.org 首次审核上传和后续 SVN `tags/0.3.2/`
必须来自同一标签提交；禁止在本地重新压缩后仍使用 `0.3.2` 名称。

GitHub Actions 完成后必须重新下载 ZIP 与校验文件，独立计算 SHA-256，并确认
发布页产物一致，再提交 WordPress.org。

## WordPress.org 资料

提交表单文案、审核说明、简体中文目录翻译底稿、图片资产和 SVN 清单统一位于
[`docs/wordpress-org-submission/`](wordpress-org-submission/README.md)。图片属于
未来 SVN 顶层 `assets/`，不属于插件 ZIP。

WordPress.org 审核通过后才向 SVN 推送功能代码。`trunk/readme.txt` Stable tag、
`tags/0.3.2/` 中的插件头版本和 GitHub Release 必须保持一致。
