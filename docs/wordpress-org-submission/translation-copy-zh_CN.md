# Npcink Ad WordPress.org 简体中文目录文案

这份文件用于在 WordPress.org 翻译平台维护插件目录页的简体中文文案，
不是需要提交到插件 SVN `trunk/` 的额外 readme。

## 简短描述

> 使用 WordPress 区块创建、预览、排期并发布站内自有推广，默认不收集访客数据。

## 详细介绍

Npcink Ad 是一套专注、WordPress 原生的站内推广发布流程，适用于公告、
联盟营销卡片、活动素材和其他由本站自行管理的推广内容。

编辑只需创建一条“推广”，使用 WordPress 核心区块编排内容，选择它可以出现
的位置、文章或页面，通过真实主题页面预览后发布或暂停。无需再建立广告组、
广告位、跟踪对象或独立设置页面。

### 专注的发布流程

- 使用核心区块创建推广内容，包括由站点自行控制的视频区块。
- 支持手动区块、正文前、正文后、第 1–20 个段落后，以及正常文档流中的
  页面顶部或底部横栏。
- 支持全部标准文章和页面、仅文章、仅页面、指定内容，以及按核心分类或标签
  匹配文章；明确排除项始终优先。
- 支持全部设备、桌面端（782px 及以上）或移动端（781px 及以下），无需按
  User-Agent 生成不同的缓存 HTML。
- 可设置开始和停止时间。
- 可在真实页面中预览，并查看与前台展示共用的服务端资格判定结果。
- 推广列表直接显示状态、位置、内容范围、停止时间和不展示原因，并可快速暂停
  或恢复。
- 发布前检查空内容、无效目标、缺少段落锚点、不安全视频来源和错误排期。

### 默认保护隐私

Npcink Ad 不收集展示、点击、IP 地址、User-Agent、来源页面、Cookie 或访客
身份，不创建分析数据表或事件队列，也不会由插件主动请求外部 API。访客浏览器
仍可能请求站点管理员在推广内容中选择的图片或视频地址。

Npcink Ad 不是 AdSense 管理器、任意代码插入器、跟踪平台、弹窗生成器、
A/B 测试套件或同意管理工具。它专门用于通过 WordPress 原生内容和控件发布
站内自有推广。

WordPress.org 简体中文语言包提供完整的 PHP 管理界面翻译，插件包内的 JSON
目录继续提供完整的区块编辑器翻译。

## 安装步骤

1. 安装并启用 Npcink Ad。
2. 打开“Npcink Ad > 推广”，新建一条推广。
3. 使用 WordPress 区块制作内容，然后打开“编辑投放设置”。
4. 选择展示位置、内容范围、设备和可选排期。
5. 打开“真实页面预览”，确认运行判定结果。
6. 发布推广；需要暂停时将其改为草稿。

使用“手动区块”位置时，请先保存推广，再在目标页面插入“Npcink Ad 推广”
区块并选择同一条推广。短代码 `[npcink_ad promotion="ID"]` 是面向高级用户的
替代入口。

## 常见问题

### Npcink Ad 会跟踪访客吗？

不会。它不收集展示或点击分析，不设置访客 Cookie，不持久化页面横栏的关闭
状态，也不会由插件主动请求外部 API。浏览器只会请求站点管理员选择的推广媒体。

### “手动区块”会自动插入推广吗？

不会。请保存推广，在需要展示的位置插入“Npcink Ad 推广”区块，并选择同一条
推广。缺少区块的提示是建议性检查，因为模板、同步样板或短代码也可能提供入口；
请通过“真实页面预览”验证目标页面。

### 页面缓存会怎样影响排期？

发布、暂停、恢复、开始和停止时间到达后，缓存页面都需要重新生成。请让缓存
有效期覆盖这些边界，或在相应时点清理受影响页面。Npcink Ad 会在 WordPress
暴露已启用的高级缓存 drop-in 时提醒管理员，但无法保证所有缓存服务都按分钟
准确切换。

### 支持哪种视频？

推广内容可以使用由站点自行控制的核心“视频”区块，来源必须是站内相对路径或
明确的 HTTP(S) 地址，并沿用同一套预览、排期、设备和位置规则。Npcink Ad
不提供 VAST/IMA、前贴片、激励视频、播放分析、弹窗视频或广告联盟播放器。

### 页面横栏会吸顶，关闭状态会被记住吗？

不会。顶部和底部横栏都位于正常文档流中；关闭按钮只在当前页面浏览期间隐藏
横栏，不使用 Cookie 或本地存储。
