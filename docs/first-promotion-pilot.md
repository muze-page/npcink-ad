# 首次成功投放试用协议

## 目的

验证一个熟悉 WordPress 基础编辑、但不了解 Npcink Ad 的用户，能否在没有
产品文档和口头步骤提示的情况下完成第一条 Promotion。自动化 E2E 证明功能
可以运行；本协议验证界面是否足够容易理解。二者不能互相替代。

## 固定环境

- 全新单站 WordPress，使用默认主题；
- 已发布一个标题为“活动页”的普通页面；
- 只启用 Npcink Ad，不预置 Promotion；
- 测试者拥有 `manage_npcink_ads`，但不展示仓库 README；
- 若启用整页缓存，必须单独记录插件、模式、TTL 和 purge 方法。

## 只提供给测试者的任务

> 请创建一条站内推广，内容为“本周活动”，让它只出现在“活动页”的正文前。
> 在真实页面中预览并发布，确认前台出现后暂停它，再恢复发布。

观察者不得解释“Promotion”“Content scope”“Real-page preview”等界面词语，
也不得指出控件位置。只有在测试者明确放弃或超过 10 分钟后才可结束任务。

## 记录项

| 项目 | 记录方式 |
| --- | --- |
| 首次前台命中时间 | 从进入 Promotion 列表到匿名页面出现内容的分钟数 |
| 是否独立完成 | 无提示完成 / 接受提示后完成 / 未完成 |
| 页面范围是否正确 | 只命中活动页，未误投其他页面 |
| 是否使用真实页面预览 | 是 / 否 |
| 是否理解暂停与恢复 | 两次前台状态均正确 / 部分正确 / 错误 |
| 首个停顿点 | 原话和所在界面，不替测试者归因 |
| 缓存影响 | 无缓存 / 命中旧缓存 / purge 后恢复 / 其他 |

## 阶段通过标准

- 完成 5 次相互独立的真实用户试用；
- 至少 4 人在 5 分钟内、无提示完成首次前台命中；
- 5 人都没有把 `selected` 误解成会自动插入手动区块；
- 5 人都能根据列表状态完成暂停和恢复；
- 每个失败都有一个可复现的界面或环境证据，而不是笼统的“不会用”。

自动化、开发者自测和 AI 子代理试用不计入这 5 次真实用户试用。没有真实用户
记录时，版本可以作为技术候选包交付，但不得宣称首次使用目标已经被用户验证。

## 清理

每场结束后删除测试 Promotion 和测试页面，清除整页缓存，并确认没有新增访客
Cookie、统计请求、option、meta schema 或自定义表。下一场必须从相同的空白
条件重新开始。
